# 🧠 Memory Arena

> Grid-based 2D головоломка по управлению оперативной памятью — размещайте фигурные блоки на сетке, освобождайте память по указателям, убирайте мусор и не допускайте утечек.

👉 **[Открыть Memory Arena](https://resenpai.github.io/memory-arena/)**

## Описание

Вы — диспетчер оперативной памяти. Программы присылают запросы на выделение (ALLOC) и освобождение (FREE) памяти в виде фигурных блоков на 2D-сетке. Размещайте блоки вручную, освобождайте по указателям, дефрагментируйте мусор и набирайте очки, чтобы пройти уровень.

### Основные механики

- **ALLOC** — размещение фигурного блока (тетромино-подобного) на сетке вручную
- **FREE** — освобождение блока по указателю (0xXXXX) с дедлайном; просрочка → мусор
- **Фигурные блоки** — 8 форм (dot, line2–4, square, L, T, Z) с вращением (R)
- **Ghost-превью** — полупрозрачная подсветка (зелёная = можно, красная = нельзя)
- **Garbage / Дефрагментация** — просроченные блоки → мусор, который нужно перетащить или убрать
- **Pointer Loss** — потеря указателя на высоких уровнях; нужно найти блок вручную
- **Стабильность** — падает при утечках и переполнении очереди; ≤ 0 = Game Over

### 5 уровней

| Уровень | Сетка | Цель | Особенности |
|---------|-------|------|-------------|
| 1. Песочница | 8×8 | 500 | 2 процесса, базовые фигуры |
| 2. Разминка | 10×10 | 1000 | 3 процесса, больше фигур |
| 3. Многозадачность | 12×12 | 2000 | 4 процесса, pointer loss 5% |
| 4. Давление | 16×16 | 3000 | 5 процессов, pointer loss 8% |
| 5. Хаос | 20×20 | 5000 | 6 процессов, pointer loss 12% |

## 🚀 Запуск локально

```bash
# Клонировать
git clone https://github.com/ReSenpai/memory-arena.git
cd memory-arena

# Установить зависимости
pnpm install

# Запустить dev-сервер
pnpm dev

# Запустить тесты
pnpm test

# Линтинг
pnpm run lint

# Сборка
pnpm run build
```

## 📁 Архитектура

```
src/
├── domain/          ← Чистая логика, 0 зависимостей
│   ├── MemoryGrid.ts      — 2D сетка: place, remove, garbage, BFS-фрагментация
│   ├── PointerRegistry.ts — реестр указателей (0xXXXX), pointer loss
│   ├── Shapes.ts          — 8 фигур, нормализация, вращение
│   ├── Scorer.ts          — подсчёт очков и стабильности
│   ├── GarbageManager.ts  — конвертация утечек → мусор
│   └── types.ts           — типы и интерфейсы
│
├── game/            ← Оркестрация
│   ├── GameSession.ts     — фасад: grid + pointers + garbage + scorer
│   ├── GameLoop.ts        — rAF loop с tick accumulator
│   ├── RequestGenerator.ts— генерация shaped ALLOC + FREE запросов
│   ├── LevelManager.ts    — конфигурация 5 уровней (grid size, target score)
│   └── SeededRandom.ts    — детерминированный LCG генератор
│
├── store/           ← Zustand
│   └── gameStore.ts       — единый store (syncFromSession)
│
├── ui/              ← React-компоненты
│   ├── App.tsx            — layout: header → stats → grid → queue
│   ├── GridCanvas.tsx     — Canvas 2D: блоки, мусор, ghost-превью
│   ├── HelpModal.tsx      — справка и правила v2
│   ├── StatsBar.tsx       — уровень, счёт, стабильность, тик
│   ├── RequestQueue.tsx   — горизонтальная очередь + таймеры FREE
│   ├── GameControls.tsx   — кнопки управления
│   ├── GameOverlay.tsx    — экран победы/поражения
│   ├── useGameLoop.ts     — хук игрового цикла (500ms тик)
│   └── ErrorNotification.tsx
│
└── __tests__/       ← 144+ теста
```

**Принципы:** Domain — чистый TS без зависимостей. Game — оркестратор. Store — тонкий клей. UI — тупые компоненты.
