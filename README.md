# 🧠 Memory Arena

> Grid-based 2D головоломка по управлению оперативной памятью — перетаскивайте фигурные блоки на сетку, освобождайте память по указателям, убирайте мусор и не допускайте утечек.

## 🎮 Играть

👉 **[Открыть Memory Arena](https://resenpai.github.io/memory-arena/)**

## Описание

Вы — диспетчер оперативной памяти. Программы присылают запросы на выделение (ALLOC) и освобождение (FREE) памяти в виде фигурных блоков на 2D-сетке. Перетаскивайте блоки мышкой, освобождайте по указателям, дефрагментируйте мусор и набирайте очки, чтобы пройти уровень.

### Основные механики

- **Drag & Drop** — зажмите карточку запроса и перетащите на сетку
- **ALLOC** — размещение фигурного блока (тетромино-подобного) на сетке перетаскиванием
- **FREE** — перетащите FREE-карточку на подсвеченный блок для освобождения
- **Фигурные блоки** — 8 форм (dot, line2–4, square, L, T, Z) с вращением (R)
- **Ghost-превью** — полупрозрачная подсветка при перетаскивании (зелёная = можно, красная = нельзя)
- **Указатели на блоках** — каждый блок отображает свой pointer (0xXXXX), при FREE целевой блок пульсирует
- **Garbage / Дефрагментация** — просроченные блоки → мусор, перетащите на свободное место
- **Pointer Loss** — потеря указателя на высоких уровнях; нужно найти блок вручную
- **Стабильность** — падает при утечках и переполнении очереди; ≤ 0 = Game Over

### Управление

| Действие | Как |
|----------|-----|
| Разместить блок | Перетащить ALLOC-карточку на сетку |
| Повернуть блок | **R** во время перетаскивания |
| Освободить блок | Перетащить FREE-карточку на целевой блок |
| Переместить мусор | Перетащить garbage-блок на сетке |
| Пауза | **P** |

### 5 уровней

| Уровень | Сетка | Цель | Особенности |
|---------|-------|------|-------------|
| 1. Stack Basics | 8×8 | 500 | 2 процесса, базовые фигуры |
| 2. Heap Growth | 10×10 | 1000 | 3 процесса, больше фигур |
| 3. Dangling Pointers | 12×12 | 2000 | 4 процесса, pointer loss 5% |
| 4. Fragmentation | 16×16 | 3000 | 5 процессов, pointer loss 8% |
| 5. Memory Chaos | 20×20 | 5000 | 6 процессов, pointer loss 12% |

## 🚀 Запуск локально

```bash
# Клонировать
git clone https://github.com/ReSenpai/memory-arena.git
cd memory-arena

# Установить зависимости
pnpm install

# Запустить dev-сервер
pnpm dev

# Запустить тесты
pnpm test

# Линтинг
pnpm run lint

# Сборка
pnpm run build
```

## 📁 Архитектура

```
src/
├── domain/          ← Чистая логика, 0 зависимостей
│   ├── MemoryGrid.ts      — 2D сетка: place, remove, garbage, BFS-фрагментация
│   ├── PointerRegistry.ts — реестр указателей (0xXXXX), pointer loss
│   ├── Shapes.ts          — 8 фигур, нормализация, вращение
│   ├── Scorer.ts          — подсчёт очков и стабильности
│   ├── GarbageManager.ts  — конвертация утечек → мусор
│   └── types.ts           — типы и интерфейсы
│
├── game/            ← Оркестрация
│   ├── GameSession.ts     — фасад: grid + pointers + garbage + scorer
│   ├── RequestGenerator.ts— генерация shaped ALLOC + FREE запросов
│   ├── LevelManager.ts    — конфигурация 5 уровней
│   └── SeededRandom.ts    — детерминированный LCG генератор
│
├── scenes/          ← Phaser scenes
│   ├── BootScene.ts       — генерация текстур и звуков
│   ├── MenuScene.ts       — главное меню с выбором уровня
│   ├── GameScene.ts       — основная игровая сцена: сетка, HUD, drag & drop
│   ├── GameOverScene.ts   — экран победы/поражения
│   ├── RequestQueueHUD.ts — горизонтальная очередь запросов
│   ├── StatsBar.ts        — панель уровня, счёта, стабильности
│   ├── DragDropManager.ts — управление drag & drop (ALLOC, FREE, Garbage)
│   └── AnimationManager.ts— визуальные эффекты (вспышки, dissolve)
│
├── assets/          ← Генерируемые ассеты
│   ├── TextureGenerator.ts— текстуры через Phaser Graphics API
│   └── SoundGenerator.ts  — звуки через Web Audio oscillators
│
└── __tests__/       ← 115 тестов
```

**Принципы:** Domain — чистый TS без зависимостей. Game — оркестратор. Scenes — Phaser 3 UI.
